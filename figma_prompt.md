# Промпт для Figma Make: Дизайн веб-приложения для перевозки автомобилей

## Контекст проекта

Разрабатывается веб-приложение для ООО «КИБЕРТРАКС» — компании по перевозке премиальных автомобилей. Приложение должно иметь современный, красивый дизайн и состоять из двух частей: пользовательской (заказ перевозки) и административной (управление заказами, тарифами, городами).

## Целевая аудитория

- **Пользователи**: Физические лица, владельцы дорогих автомобилей, желающие перевезти свой автомобиль
- **Администраторы**: Сотрудники компании, управляющие заказами и настройками системы

## Стиль дизайна

- Современный, минималистичный дизайн
- Премиальный вид (перевозка дорогих автомобилей)
- Чистые формы, достаточное белое пространство
- Профессиональная цветовая палитра (можно использовать темно-синий/черный как основной, акценты золотого/оранжевого)
- Адаптивный дизайн для десктопа и планшетов
- Использование Tailwind CSS для стилизации

## ЭКРАНЫ ПОЛЬЗОВАТЕЛЬСКОЙ ЧАСТИ

### 1. Главная страница / Лендинг

**Назначение**: Первое впечатление, информация о компании, призыв к действию

**Элементы**:
- Шапка сайта с логотипом компании "КИБЕРТРАКС"
- Навигационное меню (Главная, О компании, Контакты)
- Кнопка "Заказать перевозку" (CTA)
- Hero-секция с заголовком и подзаголовком о перевозке премиальных автомобилей
- Блок преимуществ (закрытые автовозы, безопасность, опыт)
- Блок популярных маршрутов (Бишкек-Москва, Москва-Сочи)
- Футер с контактами

**Особенности**:
- Визуально привлекательный, создает доверие
- Явный призыв к действию "Заказать перевозку"

### 2. Регистрация / Авторизация

**Назначение**: Регистрация нового пользователя или вход существующего

**Элементы**:
- Форма регистрации:
  - Поле "ФИО" (обязательное)
  - Поле "Телефон" (обязательное, маска ввода)
  - Кнопка "Зарегистрироваться"
- Форма авторизации (если пользователь уже зарегистрирован):
  - Поле "Телефон"
  - Поле "Пароль" (или можно использовать только телефон для упрощения)
  - Кнопка "Войти"
- Переключение между регистрацией и входом
- Валидация полей (визуальная индикация ошибок)

**Особенности**:
- Простая, быстрая регистрация (минимум полей)
- Можно объединить в один экран с табами

---

### 3. Форма заказа перевозки

**Назначение**: Ввод данных для расчета стоимости перевозки

**Элементы**:
- Заголовок "Заказ перевозки автомобиля"
- Поле "Марка и модель автомобиля":
  - Текстовое поле с автодополнением
  - Выпадающий список популярных марок/моделей (BMW, Mercedes, Audi, Lexus, Porsche и т.д.)
  - Возможность свободного ввода
- Поле "Город отправления":
  - Выпадающий список доступных городов (из админки)
  - Поиск по городам
- Поле "Город назначения":
  - Выпадающий список доступных городов
  - Поиск по городам
- Поле "Дата начала перевозки":
  - Календарь выбора даты
  - Минимальная дата = сегодня
- Кнопка "Рассчитать стоимость"
- Индикатор загрузки при расчете

**Особенности**:
- Пошаговая форма или все поля на одном экране
- Валидация: города должны быть разными
- Визуальная обратная связь при заполнении

---

### 4. Экран расчета стоимости

**Назначение**: Отображение рассчитанной стоимости и параметров перевозки

**Элементы**:
- Информация о заказе:
  - Марка и модель автомобиля
  - Маршрут: "Город А → Город Б"
  - Дата начала перевозки
- Блок расчета стоимости:
  - Расстояние (км)
  - Примененный тариф (руб/км)
  - Стоимость перевозки (крупно, выделено)
  - Стоимость страховки (10% от стоимости перевозки)
  - **Итого к оплате** (крупно, акцент)
- Блок информации о доставке:
  - Время в пути: X часов / Y дней
  - Дата прибытия: DD.MM.YYYY
- Кнопка "Подтвердить заказ"
- Кнопка "Изменить параметры" (вернуться назад)

**Особенности**:
- Четкая визуализация итоговой суммы
- Информация о сроках доставки
- Возможность вернуться и изменить параметры

### 5. Экран подтверждения заказа

**Назначение**: Финальное подтверждение перед оплатой

**Элементы**:
- Заголовок "Подтвердите заказ"
- Карточка с полной информацией о заказе:
  - Данные клиента (ФИО, телефон)
  - Автомобиль (марка и модель)
  - Маршрут
  - Даты
  - Стоимость (разбивка)
  - Итого
- Чекбокс "Я согласен с условиями перевозки"
- Кнопка "Перейти к оплате"
- Кнопка "Отменить заказ"

**Особенности**:
- Финальная проверка всех данных
- Четкий призыв к оплате

---

### 6. Экран оплаты (симулятор)

**Назначение**: Симуляция оплаты заказа

**Элементы**:
- Заголовок "Оплата заказа"
- Информация о сумме к оплате (крупно)
- Форма ввода данных карты:
  - Поле "Номер карты" (16 цифр, с пробелами каждые 4 цифры)
  - Поле "Срок действия" (MM/YY)
  - Поле "CVV" (3 цифры)
  - Поле "Имя держателя карты"
- Индикатор платежной системы:
  - Логотип Visa (начинается с 4)
  - Логотип Mastercard (начинается с 5)
  - Логотип МИР (начинается с 2)
  - Автоматическое определение по первой цифре
- Кнопка "Оплатить" (основная, крупная)
- Кнопка "Отменить оплату"
- Индикатор безопасности (замочек, SSL)

**Особенности**:
- Современный дизайн формы карты
- Визуальная обратная связь при вводе
- Автоматическое определение платежной системы
- Все оплаты успешны (не нужны сообщения об ошибках)

### 7. Экран успешной оплаты

**Назначение**: Подтверждение успешной оплаты

**Элементы**:
- Иконка успеха (галочка, зеленый цвет)
- Заголовок "Оплата успешно выполнена"
- Номер заказа
- Информация о заказе (кратко)
- Кнопка "Вернуться на главную"
- Кнопка "Мои заказы" (если будет реализовано)

**Особенности**:
- Позитивная обратная связь
- Четкая информация о следующем шаге

### 9. Главная страница админки (Реестр заказов)

**Назначение**: Просмотр всех заказов с фильтрацией и сортировкой

**Элементы**:
- Шапка с навигацией:
  - "Реестр заказов" (активно)
  - "Города"
  - "Тарифы"
  - Кнопка "Выйти"
- Блок фильтров:
  - Поле "Дата" (календарь выбора даты)
  - Поле "Период" (выбор диапазона дат)
  - Поле "Город отправления" (выпадающий список)
  - Поле "Город назначения" (выпадающий список)
  - Поле "Статус оплаты" (выпадающий: Все / Оплачено / Не оплачено)
  - Кнопка "Применить фильтры"
  - Кнопка "Сбросить"
- Блок сортировки:
  - Сортировка по дате (↑↓)
  - Сортировка по стоимости (↑↓)
- Таблица заказов:
  - Колонки:
    - Номер заказа
    - Клиент (ФИО)
    - Автомобиль
    - Маршрут
    - Дата начала
    - Дата прибытия
    - Стоимость
    - Статус оплаты
    - Статус перевозки (цветовая индикация)
  - Цветовая индикация строк:
    - **Зеленый**: Заказ еще не уехал (дата начала > выбранной даты)
    - **Желтый**: Заказ в пути (дата начала ≤ выбранной даты ≤ дата прибытия)
    - **Синий**: Заказ прибыл (дата прибытия < выбранной даты)
  - Клик по строке открывает детальную информацию
- Пагинация (если заказов много)

**Особенности**:
- Удобная фильтрация для быстрого поиска
- Цветовая индикация статусов перевозки
- Выделение неоплаченных заказов (важно для обзвона)

**Элементы**:

- Таблица городов:
  - Колонки:

    - Действия (Редактировать / Удалить)
- Модальное окно "Добавить/Редактировать город":
  - Поле "Название города"
  - Кнопка "Сохранить"
  - Кнопка "Отмена"

**Особенности**:
- Простой интерфейс для управления справочником
- Валидация: нельзя удалить город, если есть заказы с этим городом

### 12. Управление тарифами

**Назначение**: Настройка тарифов по месяцам

**Элементы**:
- Заголовок "Управление тарифами"
- Таблица тарифов:
  - Колонки:
    - Месяц
    - Тариф до 1000 км (руб/км)
    - Тариф свыше 1000 км (руб/км)
    - Действия (Редактировать)
- Кнопка "Редактировать" открывает модальное окно:
  - Поле "Месяц" (выбор из списка или календарь)
  - Поле "Тариф до 1000 км" (руб/км)
  - Поле "Тариф свыше 1000 км" (руб/км)
  - Кнопка "Сохранить"
  - Кнопка "Отмена"
- Информация о фиксированных маршрутах:
  - Блок с информацией:
    - Москва — Сочи: 200 000 руб (фиксированная)
    - Бишкек — Москва: 350 000 руб (фиксированная)
  - Примечание: "Эти маршруты не зависят от тарифов"

**Особенности**:
- Удобное редактирование тарифов по месяцам
- Визуальное разделение фиксированных маршрутов

## ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ К ДИЗАЙНУ

### Компоненты и паттерны

- **Формы**: Современные поля ввода с плавающими лейблами или стандартными лейблами
- **Кнопки**: Четкая иерархия (primary, secondary, danger)
- **Карточки**: Для группировки информации
- **Модальные окна**: Для форм добавления/редактирования
- **Таблицы**: Адаптивные, с возможностью сортировки
- **Индикаторы**: Цветовые бейджи для статусов
- **Иконки**: Минималистичные, современные

### Цветовая палитра (рекомендация)

- **Основной**: Темно-синий (#1E3A5F) или черный (#000000)
- **Акцент**: Золотой (#D4AF37) или оранжевый (#FF6B35)
- **Успех**: Зеленый (#10B981)
- **Предупреждение**: Желтый (#F59E0B)
- **Информация**: Синий (#3B82F6)
- **Ошибка**: Красный (#EF4444)
- **Фон**: Белый (#FFFFFF) или светло-серый (#F9FAFB)
- **Текст**: Темно-серый (#1F2937) на светлом фоне, белый на темном

### Типографика

- **Заголовки**: Крупные, жирные (700-800)
- **Текст**: Читабельный, средний размер (16px базовый)
- **Акценты**: Выделение важных сумм, дат

### Адаптивность

- Десктоп: 1920px, 1440px, 1280px
- Планшет: 768px, 1024px
- Мобильная версия не обязательна (но можно сделать базовую)

### Интерактивность

- Hover-эффекты на кнопках и ссылках
- Плавные переходы (transitions)
- Визуальная обратная связь при действиях
- Индикаторы загрузки

## СПЕЦИАЛЬНЫЕ ТРЕБОВАНИЯ

### Цветовая индикация статусов перевозки

- **Зеленый фон/рамка**: Заказ еще не уехал
- **Желтый фон/рамка**: Заказ в пути
- **Синий фон/рамка**: Заказ прибыл

### Определение платежной системы

- Визуальная индикация логотипа платежной системы при вводе номера карты
- Автоматическое изменение при изменении первой цифры

### Валидация форм

- Визуальная индикация ошибок (красная рамка, текст ошибки)
- Блокировка кнопок при невалидных данных
- Подсказки при наведении
